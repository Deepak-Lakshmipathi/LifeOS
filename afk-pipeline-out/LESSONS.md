- 2026-07-20 (#120): a mount/unmount setState test asserting on `result.current` after `unmount()` is VACUOUS — React 18 silently no-ops setState on an already-unmounted fiber (no throw, no re-render), so the assertion passes with OR without the guard. The ponytail-review reviewer caught it by MUTATION-TESTING: strip the guard, rerun, test still green = test proves nothing. Fix: assert on an observable side-effect that sits AFTER the guard (here `console.error` on the `.catch` path). Always mutation-test a regression test for a race/guard bug — a green test that also passes with the fix reverted is worthless. Do this yourself before dispatching review; it's ~2 min.

One shared `mountedRef` guarding refresh() + both mount continuations is a smaller, more correct diff than an effect-local `cancelled` flag that only covers the mount path. Grep the whole file for the same pattern before calling a leak fixed.
