The one job

Prove the whole pipe with the smallest possible thing — a task that lives, gets done, and stays done.

No projects. No folders. No priority. No sync. No settings. If the core loop doesn't feel instant and trustworthy here, nothing built on top of it will.

What this slice pierces, top to bottom
UI
React + Tailwind + spring motion
List, add field, tap-to-complete with a haptic pop + spring.
DATA
Dexie / IndexedDB
Tasks persist locally. Survive reload, restart, offline.
SHELL
Installable PWA
Service worker caches the app. Installs on Windows + Android. Works with no network.
SEAM
Sync seam — scaffolding only
A SyncProvider interface with a no-op LocalOnly behind it. No backend, no sync fields yet. The call site exists so a later slice swaps the body, not the wiring.

How it gets built, inside the slice
1
Scaffold
Vite + React + TS + Tailwind. vite-plugin-pwa wired. Empty installable shell that says hello offline.
2
Persist
Dexie table tasks. Behind a SyncProvider seam (LocalOnly no-op). Reactive read into the UI.
3
Loop
Add, list, toggle done, delete. The whole product is this loop — make it instant.
4
Polish
SF type, generous whitespace, spring on complete (transform/opacity), haptic on mobile, calm empty state.
5
Install
Install on the Windows laptop and the Android phone. Turn off wifi. Confirm it still adds + persists.
Slice is valid when…

Every claim below is true on a real device, with wifi off.

Add a task, kill the app, reopen — it's still there.
Installed to home screen on Android and Windows.
Works fully with no internet connection.
Completing a task feels good — spring + haptic, no lag.
Adding a task beats opening a notes app.
Sync seam exists; later slice swaps it without touching call sites.

If all six hold, the riskiest cross-platform + offline + local-persistence assumptions are proven. Everything else is additive.
