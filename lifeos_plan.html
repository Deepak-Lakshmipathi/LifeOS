Coordinator Verdict · 4-Agent Debate

LifeOS

A calm, Apple-grade place to keep track of everything you want — running offline on your phone and your laptop.

PWA · Local-first · React + TypeScript · Semi-offline sync

The Question

One user. ~17 projects across 7 life domains, each task carrying a done_when, a priority, and tags. Build a simple, elegant, Apple-feel personal tracker that works semi-offline on Android + Windows. Four specialist agents debated. Coordinator ruled below.

The Panel — Positions

The Architect

Stack & sync

Tauri (React + Rust) for one codebase, SQLite local, Git-backed sync (local commit → push to private repo). Lamport clock for conflicts; one-device-at-a-time until proven.

Strong on durability. Heavy: Rust + Tauri-Android raises the floor for a solo+AI builder.

The Interaction Designer

Apple-feel UX

Kill the grid. Home = a Today / overdue priority timeline grouped by domain. SF type, generous whitespace, priority = a single colored dot, tags on-demand. Signature detail: spring-physics drag with haptic bounce.

Adopted. Focus over dashboard. Grid demoted to a browse view.

The Data Modeler

Schema

Three entities: Task (with a cadence field), Project, RecurringCompletionLog. Habits are not a separate type — just a cadence + completion log. UUID + updated_at + soft-delete for clean merges.

Adopted whole. Lean, sync-friendly, captures habits without bloat.

The Minimalist PM

Scope & survival

Core job: add a task, check it off, see the 3 things to do today. Apps die when servant becomes secretary. Cut sync, dashboards, analytics from v1. Done = used daily for 14 days.

Adopted as the discipline. One concession: cross-device is a hard user requirement, so sync is deferred — not cut.

Coordinator Ruling

PWA, not Tauri. An installable PWA runs on Windows (Edge/Chrome) and Android from one web codebase — the fastest path for AI-assisted iteration and for the spring-animation polish that sells the Apple feel. Tauri+Rust is real power the project doesn't need yet.

Sync, simplified. Reject Git/CRDT for v1. Every record carries id (UUID), updated_at, deleted_at. Sync = push/pull the change-set with last-write-wins per record. v1 ships local-first with JSON export/import; v1.1 turns on a thin sync backend (Supabase or Turso/libSQL). "Semi-offline" satisfied, complexity contained.

UX. Designer's Today-timeline is home. The card grid survives as a secondary Domains browse — your "Life Hub" — but cards show task progress only, no separate mini-trackers in v1.

Schema. Modeler's 3 entities, verbatim. Scope. PM's ruthlessness, verbatim.

# LifeOS — v1 build kickoff

Build LifeOS: a calm, Apple-feel personal life tracker for one user
(me). Installable PWA that runs offline on Windows and Android.
Local-first now; cross-device sync comes in v1.1.

STACK
- Vite + React + TypeScript, installable PWA (vite-plugin-pwa).
- Tailwind with Apple-style tokens; Framer Motion for spring motion;
  SF-stack system fonts. Light + dark.
- IndexedDB via Dexie. No backend in v1.

SCHEMA (UUID ids; updated_at + deleted_at on every record for future
last-write-wins sync)
- Project: id, name, folder, color, sort_order, created_at,
  updated_at, deleted_at
- Task: id, project_id, folder, title, done_when, priority (0-3),
  cadence (once|daily|weekly|every-3mo|every-6mo), tags[],
  status (todo|doing|done), created_at, updated_at, completed_at,
  deleted_at
- RecurringCompletionLog: id, task_id, completed_date, created_at

SEED
- On first run, import ./seed_tasks_detailed.json (folders -> projects
  -> tasks). Idempotent. ~17 projects across 7 domains.

UX (this is the product — make it feel effortless)
- Home = "Today": tasks due/overdue + selected, grouped by domain,
  sorted priority 3->0. Empty state: "All clear today."
- Priority = one colored dot (0 gray,1 blue,2 yellow,3 red). No numbers.
- Tap circle to complete: haptic pop (mobile) + brief restrained
  celebration. Tap title -> upward sheet: title, priority, tags,
  done_when.
- Tags shown on demand, not cluttering the row.
- Second view "Domains": card grid, one card per life domain, showing
  task-progress only (no separate trackers in v1).
- Habits: cadence field; completing a recurring task writes a
  RecurringCompletionLog row; show streaks like "4 weeks running".

SCOPE
- IN: task/project CRUD, Today view, Domains grid, done_when, priority,
  tags, cadence+streaks, seed import, JSON export/import, installable
  offline PWA.
- OUT (v1): live cloud sync, calorie/workout/finance mini-trackers,
  analytics, sub-tasks, onboarding, settings.

BUILD ORDER
1 Skeleton: project + schema + seed import + plain list.
2 Core loop: add/complete/edit + Today view. Make it instant.
3 Apple polish: type, whitespace, dots, sheet, springs, haptics.
4 Habits + Domains grid.
5 PWA offline + install on Windows/Android + JSON export/import.

DONE WHEN: I use it daily for 14 days, Today earns the morning's first
tap, and adding a task beats typing it into a notes file.
